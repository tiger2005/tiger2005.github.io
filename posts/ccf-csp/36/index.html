T4 跳房子

第一步思路是交换一次操作的步骤。原先是先往前跳若干距离，然后往回退。这样的行动是很难处理的，但是如果变成先回退固定步数，然后前进若干步，那么一个点能到的地方就是一个区间。换句话说，此时第 个格子能到达的格子范围是

此时实际上已经可以通过数据结构在 的时间复杂度内求出答案了，不过可以更进一步。假设 为从第 个格子跳到第 个格子所需要的最少跳跃次数，可以发现 总是单调递增的。

回忆无边权最短路的广度搜索算法，需要每次取出一个未访问的距离最小的点进行扩展。由于 是单调递增的，因此只需要按顺序枚举下标 ，每当 被更新时，就将 能到达的区间 进行更新。另外，由于 是单调递增的，因此在一个位置上进行多次更新是没有意义的，可以维护最远可到达的位置，并只更新其右侧的部分。

最终的时间复杂度为 。一种可能的代码实现如下：

int main() {
  int n; cin >> n;
  vector<int> a(n + 1), k(n + 1);
  for (int i = 1; i <= n; i ++) cin >> a[i];
  for (int i = 1; i <= n; i ++) cin >> k[i];
  vector<int> dp(n + 1, 1e9);
  dp[1] = 0;
  int ptr = 1;
  for (int i = 1; i <= n; i ++) {
    if (dp[i] == 1e9) return puts("-1"), 0;
    int p = i - a[i], r = min(p + k[p], n);
    while (ptr < r) dp[++ ptr] = dp[i] + 1;
  }
  printf("%d", dp[n]);
}

不妨称打败第 只怪物为占领了 号位置。对 数组建立笛卡尔树，其中根结点是最大点。那么对应的，如果我们可以占领 号点，我们自然可以占领 号点子树的所有点。进一步的，为了继续占领，最容易的突破模式是占领 号点的父结点（另一边的 一定不小于父节点的 ），并顺便占领父节点子树的所有节点。

称第 只怪物的困难系数为收集完 子树内所有点的经验值后，打败父节点至少需要多少初始战斗力（也就是父节点的 减去子树的 之和）。简单分析可知 除了要不小于 之外，还需要不小于 号点和 号点到根的路径上困难系数的最大值。

简单来说，假设 号节点是 号节点的祖先（根据笛卡尔的性质，二者必然存在祖先后代关系），那么可以首先占领 号节点，然后从 开始不断占领其父节点，直到占领 号节点。在占领 号节点之前，占领父节点一定比提前占领 号节点更轻松（父节点的 更小），并且一定对之后的占领起到正向作用（占领越多，对攻击力提升越大）。这个论证同样适用于 号节点到根这部分的性质。
